## 1. SM 모듈 구축 현황 총괄

### 모듈 구성도

```
Sales & Marketing (SM)
├── 아웃바운드 (Outbound)
│   ├── 리드 스크래퍼 (7개 소스)
│   ├── AI SDR (스코어링 + 개인화)
│   ├── 콜드 이메일 (배치 발송)
│   └── 후속 관리 (자동 팔로업)
│
├── 인바운드 (Inbound)
│   ├── 이메일 답장 감지 (Gmail 폴링)
│   ├── AI 감정 분석
│   └── 문의 폼 자동화 (Playwright)
│
├── 콘텐츠 (Content)
│   ├── 5단계 LLM 파이프라인
│   ├── SNS 발행 (Blog/X/Instagram/Facebook)
│   ├── 이미지 생성
│   └── 콘텐츠 캘린더
│
└── 분석 (Analytics)
    ├── GSC (검색 노출/순위)
    ├── GA4 (트래픽/채널)
    ├── KPI 대시보드
    └── AI 인용 모니터링
```

### 기능별 완성도 평가

| 기능 | 완성도 | 신뢰도 | 판정 |
|------|--------|--------|------|

| **콜드 이메일 발송** | 70% | 중 | 정책은 있으나 워밍업 미구현 |

| **전환 추적** | 10% | 치명적 | **어트리뷰션 전무** |

## 2. 콘텐츠 제작 워크플로우 진단

### 현재 구조

```
Phase 1: Market Analyst (시장 분석)
    ↓
Phase 2: Structure Architect (구성 설계)
    ↓
Phase 3: Content Writer (본문 작성)
    ↓
Phase 4: SEO Optimizer (SEO 최적화)
    ↓
Phase 5: Quality Reviewer (품질 검수)
    ↓
[SNS 발행] → Blog / X / Instagram / Facebook
```

### 문제점 (코드 근거)

#### A. "자동 발행"으로 사용하기엔 위험

| 문제 | 코드 근거 | 심각도 |
|------|-----------|--------|
| AI 출력 JSON 파싱 실패 시 rawResponse 그대로 저장 | `content-pipeline.ts:213-216` | 높음 |
| NG 워드 체크가 단순 문자열 포함만 검사 (단어 경계 무시) | `content-pipeline.ts:258-259` | 중 |
| Phase 간 출력 검증 없음 (3→4 전달 시 구조 미검증) | 전체 파이프라인 | 중 |
| Fire-and-forget 패턴 (에러 catch 후 무시) | `content-pipeline/route.ts:25` | 높음 |
| X/Twitter 이미지 업로드 미구현 (텍스트만 발행) | `sns-publishers.ts:128-135` | 높음 |
| SNS Cron 실패 시 재시도 없이 영구 실패 처리 | `cron/sns-publish/route.ts:62-73` | 중 |
| **대시보드 UI가 하드코딩 데이터** (API 미연결) | `sm/content/page.tsx:33-119` | 치명적 |

#### B. "키워드/소재 발굴 도구"로는 즉시 활용 가능

파이프라인의 **Phase 1 (Market Analyst)** 과 **Phase 2 (Structure Architect)** 는 실제로 잘 동작한다:

- Phase 1: 시장 트렌드, 경쟁사 분석, 타겟 키워드 도출
- Phase 2: 기사 구조, 헤딩 배치, 키워드 배분 전략

이 두 단계만 분리해서 **"리서치 도구"로 전환**하면:

```
[현재] 5단계 자동 발행 (불안정)
    ↓ 전환
[제안] 2단계 리서치 → 사람이 검토 → 수동 제작/발행

Phase 1 (Market Analyst) → 키워드 리스트 + 트렌드 분석
Phase 2 (Structure Architect) → 기사 구조안 + 소재 제안
    ↓
담당자 검토 & 선별
    ↓
REbrain 2 프로세스로 본문 제작 (CLAUDE.md 기준)
    ↓
수동 발행 (품질 보장)
```

### 콘텐츠 전략 제안

| 현재 | 제안 | 이유 |
|------|------|------|
| 5단계 자동 발행 | Phase 1-2만 리서치 도구로 활용 | 품질 통제 불가능한 자동 발행은 브랜드 리스크 |
| 모든 콘텐츠 AI 생성 | AI 리서치 + 사람 제작 | REbrain 2 프로세스가 이미 있음 |
| SNS 자동 발행 | Instagram만 자동, 나머지 수동 | X 이미지 미구현, Cron 재시도 없음 |
| 대시보드 미연결 | GSC/GA4 데이터를 콘텐츠 성과 분석에 연결 | 어떤 콘텐츠가 효과 있는지 측정 |

#### 구체적 활용 시나리오

**키워드 발굴:**
- GSC 데이터에서 **높은 노출 + 낮은 CTR** 키워드 추출 → 콘텐츠 최적화 대상
- Phase 1으로 해당 키워드의 **경쟁 기사 분석** → 차별화 포인트 도출
- 월간 콘텐츠 캘린더에 반영

**소재 발굴:**
- Phase 1: "レンタカー + [지역명/계절/트렌드]" 조합으로 토픽 탐색
- Phase 2: 각 토픽의 기사 구조안 생성 → 담당자가 유망한 것 선별
- 선별된 소재를 REbrain 2 워크플로우로 제작

## 3. 아웃바운드 리드 피로도 진단

### 현재 리드 소스 현황

```
리드 소스 7개:
├── DuckDuckGo 웹 검색 (6개 고정 쿼리)
├── Homemate Research 디렉토리
├── Navitime POI (전화번호만 → 이메일 없음, 사실상 무효)
├── iタウンページ (NTT)
├── goo Townpage
├── Indeed 구인공고
└── 求人ボックス (Kyujin Box)
```

### 리드 고갈 구조 (코드 근거)

**핵심 문제: 동일 소스를 동일 쿼리로 반복 순환**

```
매 Cron 실행: 3개 현(県) × 6개 쿼리
47현 전체 순환: ~16일
순환 완료 후: 인덱스 0부터 동일 쿼리로 재시작
키워드 갱신: 47현 순환 완료 시에만 1회 (15-16일마다)
```

**피로도 관리 코드 부재:**
```
❌ "마지막 연락일로부터 N일 경과" 필터 없음
❌ "연락 횟수 N회 초과" 체크 없음
❌ "소프트 바운스" 추적 없음
❌ 리드 로테이션/휴면 로직 없음
❌ 퍼지 매칭 중복 제거 없음 (정확한 회사명 일치만)
```

**콜드 이메일 대상 선정 (auto-cold-email cron):**
```typescript
// 현재 코드: 피로도 무시, 스코어 높은 순으로만 발송
where: {
  source: { startsWith: 'scraper:' },
  email: { not: null },
  status: '未交渉',    // 미교섭 상태만
}
orderBy: { leadScore: 'desc' }
// → 연락 이력, 마지막 발송일, 바운스 여부 전혀 고려하지 않음
```

## 4. 인바운드 유도 분석 — 부족한 점

### 현재 인바운드 채널

| 채널 | 상태 | 문제 |
|------|------|------|
| 웹사이트 블로그 | 운영중 | SEO 콘텐츠 품질/양 부족 |
| 이메일 답장 | 구현됨 | **30일 간격 폴링** (실시간 아님) |
| 문의 폼 | 부분 구현 | Playwright 기반 (자사 폼이 아님) |
| 전화/SMS | 없음 | 채널 자체 부재 |
| 채팅/메신저 | 없음 | 채널 자체 부재 |
| SNS DM | 없음 | 채널 자체 부재 |
| 데모 요청 | 수동 | 자동 캡처 없음 |
| 웨비나/세미나 | 없음 | 채널 자체 부재 |
| 추천/소개 | 없음 | 프로그램 부재 |

### 인바운드 퍼널 분석

```
[현재 퍼널]

                    ┌─────────────────┐
                    │  블로그 트래픽    │  ← GSC/GA4로 측정 가능
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  ???            │  ← 전환 경로 추적 불가
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  문의/데모 요청   │  ← 수동 집계만
                    └─────────────────┘

문제: 블로그 방문 → 문의 사이의 전환 경로가 완전한 블랙박스
```

### 핵심 부족 요소

#### A. 전환 인프라 부재

| 부족 요소 | 영향 | 우선순위 |
|-----------|------|----------|
| **UTM 파라미터 미사용** | 어떤 캠페인/채널이 유효한지 측정 불가 | 최상 |
| **전환 이벤트 미설정** | GA4에서 문의 폼 제출을 전환으로 추적 안함 | 최상 |
| **CTA 없음/부족** | 블로그 → 데모 요청으로 유도하는 경로 부재 | 상 |
| **리드 마그넷 없음** | 이메일 수집을 위한 무료 자료/도구 없음 | 상 |
| **랜딩 페이지 없음** | 캠페인별 전용 랜딩 페이지 없음 | 중 |

#### B. 콘텐츠 SEO 약점

| 부족 요소 | 현재 상태 | 개선 방향 |
|-----------|-----------|-----------|
| **콘텐츠 양** | 51개 기존 블로그 | 월 4-8개 신규 필요 |
| **키워드 전략** | GSC 데이터 있으나 활용 미흡 | 높은 노출/낮은 CTR 키워드 우선 최적화 |
| **내부 링크** | 분석/관리 도구 없음 | 클러스터 구조 필요 |
| **경쟁사 분석** | 코드는 있으나 활용 안됨 | 키워드 갭 분석 정기 실행 |
| **메타 데이터** | 관리 도구 없음 | 타이틀/디스크립션 최적화 |
| **Core Web Vitals** | 추적 없음 | 페이지 속도 모니터링 |

#### C. 인바운드 리드 캡처 메커니즘 부재

현재 웹사이트(kaflixcloud.co.jp)에서 리드를 캡처하는 자동화 경로가 없다:

```
[필요한 인바운드 퍼널]

  SEO 콘텐츠 (블로그)
       │
       ▼
  CTA (데모 요청/자료 다운로드)  ← 현재 없음
       │
       ▼
  랜딩 페이지 (전환 최적화)     ← 현재 없음
       │
       ▼
  폼 제출 → Webhook → CRM      ← 현재 없음 (수동)
       │
       ▼
  자동 후속 이메일              ← 부분 구현
       │
       ▼
  영업 팀 전달                  ← 수동
```

## 5. 전략적 방향 제안

### 전체 방향: "측정 가능한 인바운드 중심 전환"

```
현재: 아웃바운드 의존 (콜드 이메일 중심, 리드 고갈)
  ↓
전환: 인바운드 기반 + 아웃바운드 보완 (측정 가능한 구조)
```

### Phase 1: 기반 정비 (즉시, 1-2주)

#### 1-1. 콘텐츠 파이프라인 → 리서치 도구 전환

```
AS-IS: 5단계 자동 발행 (불안정, 품질 불확실)
TO-BE: Phase 1-2 리서치 전용 + 사람 제작

실행:
- Phase 1-2만 실행하는 "리서치 모드" API 추가
- GSC "높은 노출 + 낮은 CTR" 키워드를 Phase 1 입력으로 자동 연결
- 월간 키워드/소재 리포트 자동 생성
```

## 7. 핵심 요약

### 진단

| 영역 | 현재 상태 | 핵심 문제 |
|------|-----------|-----------|
| **콘텐츠** | 파이프라인 구현됨, UI 미연결 | 자동 발행은 위험, 리서치 도구로 전환 필요 |
| **아웃바운드** | 리드 고갈, 피로도 관리 없음 | 동일 소스 반복 순환, 신규 소스 필요 |
| **인바운드** | 채널 자체 부재 | CTA/리드마그넷/전환추적 전무 |
| **분석** | GSC/GA4 수집만 | 어트리뷰션/퍼널 측정 불가 |

### 전략 방향

1. **콘텐츠**: 자동 발행 중단 → **키워드/소재 발굴 도구**로 전환, 제작은 REbrain 2
2. **아웃바운드**: 리드 피로도 관리 + 쿼리 다각화 + Google Maps/협회 DB 등 **신규 소스 확보**
3. **인바운드**: 블로그 SEO 강화 + CTA 배치 + **리드 마그넷**(무료 자료) → 이메일 수집 → 너처링
4. **분석**: UTM + GA4 전환 이벤트 → **측정 가능한 마케팅** 구조 구축

### 가장 시급한 3가지

| # | 과제 | 이유 |
|---|------|------|
| **1** | UTM + 전환 추적 설정 | 현재 아무것도 측정 못함. 측정 없이 최적화 불가 |
| **2** | 리드 피로도 관리 | 같은 리드에 반복 발송 → 스팸 신고 위험 → 도메인 평판 하락 |
| **3** | 블로그 CTA + 리드 마그넷 | 트래픽이 있어도 전환 장치가 없으면 인바운드 리드 0 |
